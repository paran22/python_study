## `.env` 파일은 어디에 두어야 할까?

### 설정 파일은 프로젝트 최상위(루트)에 둔다

프로젝트를 만들면 여러 가지 설정 파일이 필요합니다.
`.env`, `.gitignore`, `package.json` 같은 파일들이 대표적입니다.

이런 설정 파일들은 **프로젝트 폴더의 가장 바깥(루트)**에 위치시키는 것이 일반적입니다.
그래야 프로젝트 전체에서 해당 설정을 인식할 수 있기 때문입니다.

```
project-A/
├── .env              ← 루트에 위치
├── .gitignore        ← 루트에 위치
├── package.json      ← 루트에 위치
├── app.py
├── routes/
│   └── users.py
└── utils/
    └── helper.py
```

---

### `.env`는 같은 프로젝트 안에서만 적용된다

`.env` 파일을 프로젝트 루트에 두면,
**그 프로젝트의 하위 폴더까지 포함해서** 환경변수가 적용됩니다.

하지만 **다른 프로젝트에는 적용되지 않습니다.**

```
Desktop/
├── project-A/
│   ├── .env              ← 이 .env는 project-A 안에서만 적용
│   ├── app.py            ← ✅ 적용됨
│   ├── routes/
│   │   └── users.py      ← ✅ 적용됨 (하위 폴더)
│   └── utils/
│       └── helper.py     ← ✅ 적용됨 (하위 폴더)
│
├── project-B/
│   ├── .env              ← 이 .env는 project-B 안에서만 적용
│   ├── index.js          ← ✅ 적용됨
│   └── src/
│       └── app.js        ← ✅ 적용됨 (하위 폴더)
│
└── project-C/
    ├── index.js          ← ❌ .env가 없어서 환경변수 사용 불가
    └── src/
        └── app.js        ← ❌ 마찬가지로 사용 불가
```

---

### 정리

| 범위 | 적용 여부 |
|------|-----------|
| 같은 프로젝트 루트 | ✅ 적용 |
| 같은 프로젝트 하위 폴더 | ✅ 적용 |
| 다른 프로젝트 (형제 폴더) | ❌ 적용 안 됨 |
| 상위 폴더 | ❌ 적용 안 됨 |

> `.env`를 포함한 설정 파일은 **프로젝트 루트에 두는 것이 기본**이며,
> 그 설정은 **해당 프로젝트 안에서만** 효력을 가집니다.
