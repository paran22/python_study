### [1차시] 파이썬의 두 기둥: 변수와 함수로 '프롬프트 자판기' 만들기

> 💡 **LLM Point**: 코딩의 기본 규칙을 가볍게 익힌 후, 프로그래밍에서 가장 중요한 '변수'와 '함수'의 개념을 확실히 잡습니다. 연습장에서 프롬프트 재료를 조립해 보고, 로컬 환경에서 나만의 '프롬프트 생성기' 파일로 완성해 봅니다.

#### ⏱️ [20분] 비전공자를 위한 코딩 첫걸음 오리엔테이션 (OT)

- **[PPT]** 프로그래밍 언어란 무엇인가?
  - 컴퓨터는 우리 말을 못 알아듣는다 — 컴퓨터가 이해하는 언어로 '번역'해서 명령을 내리는 도구가 바로 프로그래밍 언어
  - 언어도 종류가 다양하다: 웹을 만드는 JavaScript, 앱을 만드는 Swift/Kotlin, 데이터·AI에 강한 Python 등 — 목적에 따라 쓰는 언어가 다름
  - **라이브러리와 프레임워크**: 다른 개발자가 미리 만들어 둔 코드 묶음. 라이브러리는 필요한 도구를 골라 쓰는 공구함, 프레임워크는 뼈대까지 잡아주는 조립 키트 — 지금은 둘 다 **"남이 만든 도구 모음을 가져다 쓰는 것"** 정도로 이해하면 충분!
  - 파이썬이 AI/데이터 분야의 공용어가 된 이유: 문법이 영어 문장처럼 읽히고, 이런 도구 모음이 이미 잔뜩 만들어져 있기 때문
    ```python
    # "fruits 리스트 안에 'apple'이 있으면, '있다!'를 출력해라"
    if "apple" in fruits:
        print("있다!")
    ```
    ```python
    # "에러가 나면 → '다시 시도하세요'를 출력해라"
    try:
        result = search("AI 뉴스")
    except Error:
        print("다시 시도하세요")
    ```
    → 코드를 모르는 사람도 영어 단어만 알면 대충 읽힌다는 것을 강조

- **[PPT]** 파이썬으로 할 수 있는 것들 (우리 스터디와 연결되는 것 위주)
  - **자동화**: 반복 업무(파일 정리, 엑셀 처리, 이메일 발송 등)를 코드 한 줄로 처리
  - **데이터 수집·정리**: 웹에서 정보를 긁어오거나(크롤링), API로 데이터를 받아 원하는 형태로 가공
  - **LLM/AI 연동**: ChatGPT·Gemini 같은 AI 모델에 파이썬 코드로 프롬프트를 보내고 결과를 받아올 수 있음
  - **에이전트 만들기**: 검색 → AI 요약 → 결과 저장처럼, 여러 작업을 연결해 '내 대신 일하는 AI 비서' 구축
  - (참고) 파이썬이 잘 안 쓰이는 영역: 모바일 앱, 화려한 웹 프론트엔드 — 이쪽은 Swift/Kotlin, JavaScript 등 더 적합한 언어가 따로 있음. 우리 목적엔 필요 없으니 신경 쓰지 않아도 OK

  - **IDE란?** — 코드를 작성하고 실행하는 전용 편집기 (Word가 문서 작성 도구라면, IDE는 코드 작성 도구)
  - **온라인 환경** — 설치 없이 브라우저에서 바로 실행 가능, 가볍게 테스트해보기 좋음

- **[PPT]** 프로그래밍 파일의 정체: `.py` vs `.ipynb` 차이 이해하기
  - **`.py` (스크립트 파일)**
    - 위에서 아래로 전체를 한 번에 순서대로 실행하는 파일
    - 완성된 코드를 터미널에서 `python main.py` 한 줄로 돌리는 방식
    - 로컬 IDE에서 작성하고 실행
  - **`.ipynb` (노트북 파일)**
    - 코드를 '셀(cell)' 단위로 쪼개서 한 블록씩 실행하고 결과를 바로 확인할 수 있는 파일
    - 중간 결과를 눈으로 보면서 실험하기 좋음 (API 응답 구조 확인, 문법 테스트 등)
    - 노트북에서는 셀의 마지막 줄 결과가 자동으로 출력됨 — `print` 없이도 값을 바로 확인할 수 있음
    - Google Colab 등 노트북 환경에서 사용

- **[Colab]** 파이썬의 마음 읽기: 위에서 아래로 순차 실행, 대소문자 엄격히 구분 (`Apple`과 `apple`은 남남)
- **[Colab]** 따옴표(`" "`)와 괄호의 비밀: 따옴표가 있으면 글자(데이터), 없으면 명령어! 괄호는 무조건 짝 맞추기
- **[Colab]** 띄어쓰기 vs 들여쓰기: 띄어쓰기는 자유지만, 들여쓰기(Tab)는 파이썬에서 생명이다!
- **[Colab]** 주석과 에러: `#`은 컴퓨터가 무시하는 내 메모. 일부러 에러 내보기 _(에러는 고장 난 게 아니라 파이썬이 애교 부리는 것! 쫄지 않기 마인드셋)_

- **[핵심 실습] [Cursor]** 가상환경(`venv`) 세팅
  - `python -m venv .venv` → 가상환경 생성
  - `source .venv/bin/activate` (Mac) / `.venv\Scripts\activate` (Windows) → 활성화

- **[Cursor]** `pip`으로 패키지 설치하기
  - `pip install requests` — 남이 만든 도구(패키지)를 설치하는 명령어
  - 패키지 버전 지정: `pip install requests==2.31.0` — 특정 버전을 설치할 수 있음
  - 버전이 중요한 이유: 같은 패키지라도 버전에 따라 동작이 달라질 수 있고, 팀원 간 버전이 다르면 "내 컴퓨터에선 되는데?" 문제가 발생
- **[Cursor]** `requirements.txt` 소개
  - `pip freeze > requirements.txt` — 설치된 패키지와 **버전 번호**를 함께 저장
  - 다른 사람이 `pip install -r requirements.txt` 한 줄로 동일한 버전의 환경을 복원할 수 있음

#### ⏱️ [15분] 1차시 복습 — 변수, f-string, 함수, 파일 분리와 `import`

- **[Colab]** 파일별 스코프 분리: 파일이 다르면 변수와 함수도 서로 보이지 않는다
- **[Colab]** `import`로 다른 파일의 함수 가져오기

#### ⏱️ [20분] 리스트와 딕셔너리 기초

- **[Colab]** 리스트(List): 순서가 있는 데이터 묶음
  - 생성: `fruits = ["사과", "바나나", "딸기"]`
  - 인덱스로 꺼내기: `fruits[0]` → `"사과"` (0번부터 시작!)
  - 없는 인덱스 접근 시 `IndexError` 체험
  - 길이 확인: `len(fruits)`, 추가: `.append()`, 슬라이싱: `fruits[:2]`
- **[Colab]** 딕셔너리(Dictionary): 이름표(키)로 찾는 데이터 묶음
  - 생성: `person = {"name": "가은", "age": 25, "city": "서울"}`
  - 키로 값 꺼내기: `person["name"]` → `"가은"`
  - 없는 키 접근 시 `KeyError` 체험
  - 키 목록 확인: `.keys()`

#### ⏱️ [20분] 중첩 구조 파싱 (LLM API 응답 미리보기)

- **[Colab]** 리스트 안에 딕셔너리: 가장 흔한 패턴
  - `movies[0]["title"]` — 인덱스로 딕셔너리를 꺼낸 뒤, 키로 값에 접근
- **[Colab]** 딕셔너리 안에 리스트: `user["interests"][0]`
- **[Colab]** LLM API 응답과 비슷한 구조를 단계별로 파싱하는 연습
  - `api_response["choices"][0]["message"]["content"]`
  - 한 번에 쓰지 말고, 바깥부터 한 단계씩 `print`하며 접근하는 습관 들이기
- **[Colab]** 검색 결과 형태의 중첩 데이터에서 원하는 값 꺼내기 연습

#### ⏱️ [20분] 반복문 (`for`)

- **[Colab]** 리스트의 항목을 하나씩 꺼내기: `for fruit in fruits:`
- **[Colab]** `range`와 `len`을 활용한 인덱스 기반 반복
- **[Colab]** `enumerate`: 인덱스와 값을 동시에 꺼내는 방법
- **[Colab]** 리스트 안 딕셔너리를 순회하며 특정 필드 출력하기
  - `for movie in movies:` → `print(movie["title"])`

#### ⏱️ [20분] 조건문 (`if`)

- **[Colab]** 기본 `if` / `elif` / `else` 구문
- **[Colab]** 문자열 안에 특정 단어가 포함되어 있는지 확인: `if "키워드" in text:`
- **[Colab]** `if`로 에러 미리 막기 (예외처리 패턴)
  - 딕셔너리에 키가 있는지 확인: `if "email" in person:`
  - 리스트가 비어있는지 확인: `if len(results) > 0:`
  - 값이 `None`인지 확인: `if document is not None:`
  - API 응답에서 값이 빠져있거나 비어있는 경우가 자주 있어서 이 패턴이 중요

#### ⏱️ [25분] 실제 데이터셋 불러오기와 종합 실습

- **[Colab]** 파싱 연습: `dataset[0]["document"]`로 리뷰 텍스트 꺼내기, 여러 건을 리스트로 모으기
- **[Colab]** `for` + `if` 조합: 긍정 리뷰만 10개 출력, 특정 키워드가 포함된 리뷰 찾기
